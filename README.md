# Analysing-E-Learning-Platform-Purchases-using-MySQL
## Project Objective:
This project involved designing and implementing a relational database to track an online learning platform’s sale, using SQL normalization and Foreign Key constraints to link learners, courses, and purchases.
## Dataset Description: 
The database consists of three interrelated tables designed to eliminate data redundancy and ensure referential integrity.
1. Learner’s table: This table stores the primary identity of the students.
2. Courses table: This table acts as the product catalog for the platform.
3. Purchases table: The "Bridge Table" that records every transaction. It connects the Who to the What.

## Insights Arrived:
1. Advanced SQL is currently our #1 bestseller, followed closely by Python Basics.

2. Our most successful students aren't just taking one class; they are "Skill-Jumpers" who buy courses from different categories (like Design and Programming) to build a full career.
   
3. Since students are buying Digital Marketing and Machine Learning at the same rate, we can expect a growing trend in " Digital and Data-Driven" careers for the next quarter.
   
4. We should create "Starter Bundles" (combining a top-seller with a zero-sale course) to help our underperforming classes like Tableau for data viz and React framework finally get their first students.
	
5. Our financial analysis shows that India, Canada, and Russia are the top three revenue-generating markets. These regions contribute the highest total sales volume, making them the primary drivers of our platform's growth.

## Conclusion:

This project successfully organized a messy list of sales into a clean, professional relational database that is easy to manage and grow. By using SQL analysis, we discovered exactly which courses are favourites and which ones need better marketing to help the business make smarter decisions.
